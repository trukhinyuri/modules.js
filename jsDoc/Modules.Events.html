<static> addContextListener(target, type, listener, context, useCapture)

Add event listener with bind context. Bypass problems where it's unclear what this will be, depending on the context from which function was called
Parameters:
Name Type Argument Default Description
target HTMLElement Any html element
type String The event type for which the user is registering
listener EventListener The listener parameter takes an interface implemented by the user which contains the methods to be called when the event occurs.
context object <optional>
 "this" Context scope for this inside listener
useCapture boolean <optional>
 "false" If true, useCapture indicates that the user wishes to initiate capture. After initiating capture, all events of the specified type will be dispatched to the registered EventListener before being dispatched to any EventTargets beneath them in the tree. Events which are bubbling upward through the tree will not trigger an EventListener designated to use capture. Event phases: capture -> target -> bubble.

<static> addDocumentListener(type, listener, useCapture)

Add event listener for document target
Parameters:
Name Type Argument Default Description
type String The event type for which the user is removing
listener EventListener The listener parameter takes an interface implemented by the user which contains the methods to be called when the event occurs.
useCapture boolean <optional>
 "false" If true, useCapture indicates that the user wishes to initiate capture. After initiating capture, all events of the specified type will be dispatched to the registered EventListener before being dispatched to any EventTargets beneath them in the tree. If a listener was registered twice, one with capture and one without, each must be removed separately.
